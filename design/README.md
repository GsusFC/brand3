# Design

Esta carpeta concentra la base de diseño y desarrollo frontend de Brand3.

## Archivos

- [`DESIGN.md`](/Users/gsus/brand3-scoring/design/DESIGN.md)
  - Documento maestro de producto visual, páginas, componentes y tono.
- [`FRONTEND_PLAN.md`](/Users/gsus/brand3-scoring/design/FRONTEND_PLAN.md)
  - Plan de implementación frontend por fases, con prioridades y dependencias.
- [`RESPONSIVE.md`](/Users/gsus/brand3-scoring/design/RESPONSIVE.md)
  - Breakpoints, comportamiento responsive por página y transformación de tablas/layouts.
- [`COMPONENTS.md`](/Users/gsus/brand3-scoring/design/COMPONENTS.md)
  - Inventario de componentes, variantes, propósito y reglas de composición.
- [`TOKENS.css`](/Users/gsus/brand3-scoring/design/TOKENS.css)
  - Variables CSS base para color, tipografía, spacing, radios y estados.

## Uso recomendado

1. Definir interfaz y jerarquía visual en [`DESIGN.md`](/Users/gsus/brand3-scoring/design/DESIGN.md).
2. Diseñar pantallas y componentes usando [`TOKENS.css`](/Users/gsus/brand3-scoring/design/TOKENS.css) como base.
3. Definir comportamiento responsive con [`RESPONSIVE.md`](/Users/gsus/brand3-scoring/design/RESPONSIVE.md).
4. Definir sistema de componentes en [`COMPONENTS.md`](/Users/gsus/brand3-scoring/design/COMPONENTS.md).
5. Implementar por fases siguiendo [`FRONTEND_PLAN.md`](/Users/gsus/brand3-scoring/design/FRONTEND_PLAN.md).

## Regla

Frontend de Brand3 no debe convertirse en un dashboard SaaS genérico.

La app tiene que sentirse:

- brand-first
- editorial
- operativa
- precisa

No:

- plantillero
- brillante
- hype
- recargado
